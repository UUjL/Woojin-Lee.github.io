Notaker

AI-Powered Notes from Audio & Transcripts

Sep 2025 — Apr 2026 · Team project (CS 4500)

Python FastAPI Django Electron React Next.js Whisper TypeScript

Overview

Notaker is an AI-powered note-taking application that helps students capture and organize lecture content more effectively. The system converts audio recordings into structured Markdown notes and supports attachments such as PDF slides and images. By generating organized notes automatically, students can focus on listening and understanding in class instead of writing everything by hand.

Our goal is to turn lecture content into clear, searchable study materials that are easy to review, edit, and reuse. Notaker ships as a desktop app and a web app.

Problem

Taking useful notes during fast-paced lectures is hard. Students often miss details, and handwritten notes are difficult to organize and search later. Transcription tools produce raw text but do not structure information in a meaningful way. Students need a system that organizes lecture content while preserving information from both audio and visual materials.

Solution

Notaker uses AI to generate structured notes from audio recordings and uploaded documents. Users can record or upload lecture audio, generate notes, attach PDFs and images, organize content in folders, edit with a custom Markdown editor, and restore deleted notes from a recycle system.

  • Upload or record lecture audio
  • Automatically generate structured Markdown notes
  • Attach PDFs, images, and other supporting files
  • Organize notes in folders with search and embeddings
  • Edit notes with live preview, autosave, and version-friendly workflows

Key features

  • AI-generated structured notes from audio
  • Custom Markdown editor with live preview and autosave
  • File and folder organization
  • Attachments for audio, PDFs, and images
  • Recycle and restore for deleted notes
  • Local transcription with Whisper
  • Cross-platform desktop app (Windows, macOS, Linux)
  • Web app (Next.js) for browser-based access

Architecture

Notaker uses a hybrid local + cloud design: responsive processing on the user's device, with cloud services for auth and AI-backed features.

  • Desktop (Electron + React): Main UI for recording, editing, and organizing notes. Talks to a local FastAPI service for proxying and local features.
  • Local FastAPI: Whisper speech-to-text, SQLite and file sync, attachments, translation, embedding search, and RAG pipelines exposed as unified APIs.
  • Cloud (Django): JWT authentication, API endpoints, and integration with AI services for note generation, study tools, and related features.
  • Web (Next.js): Browser client connected to Django for auth and cloud-backed functionality.

Architecture diagram and full design documentation are maintained in our project repository.

  • Progress tracking, input validation, and consistent API response schemas
  • Integration with Django services and the local proxy layer used by the Electron client

How it works

  1. Log in — Required for AI features such as note generation.
  2. Record or upload audio — Capture a lecture in the app or upload a file.
  3. Review transcription — Speech is converted to text; optional translation appears alongside.
  4. Generate notes — Structured Markdown is created from the transcript and any attached PDFs or images; embeddings are stored locally and in the database.
  5. Edit and save — Notes are saved as Markdown with autosave; rename, organize, and use recycle/restore as needed.
  6. Study tools (optional) — AI chat, quizzes, and flashcards from the note menu.
